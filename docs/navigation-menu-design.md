# 전역 네비게이션 메뉴 디자인

## 📋 요구사항

**현재 상황:**
- 홈 페이지에만 모든 카테고리와 도구 목록이 표시됨
- 각 도구 페이지(예: 룰렛, JSON Parser)에서는 다른 도구로 이동하려면 뒤로가기 → 홈 → 원하는 도구 클릭 필요

**목표:**
- 모든 페이지에서 다른 카테고리/도구로 빠르게 이동할 수 있는 네비게이션 메뉴 추가
- 어느 페이지에 있든 전체 도구 목록에 쉽게 접근 가능

## 🎨 디자인 옵션

### 옵션 1: 사이드바 네비게이션 (추천 ⭐)

```
┌─────────────┬──────────────────────────┐
│             │ Header                   │
│  Sidebar    ├──────────────────────────┤
│             │                          │
│ 🏠 홈       │                          │
│             │   Main Content           │
│ 🎮 게임     │   (도구 페이지)          │
│  · 룰렛     │                          │
│  · 사다리   │                          │
│  · 숫자뽑기 │                          │
│             │                          │
│ 💻 개발자   │                          │
│  · JSON     │                          │
│  · UUID     │                          │
│             │                          │
│ 🔄 변환     │                          │
│ 📝 텍스트   │                          │
│ 🧮 계산기   │                          │
└─────────────┴──────────────────────────┘
```

**장점:**
- 항상 보이므로 네비게이션이 쉬움
- 많은 메뉴를 계층적으로 표시 가능
- 현재 위치를 명확히 표시 가능

**단점:**
- 콘텐츠 영역이 좁아짐
- 모바일에서는 접었다 펼쳤다 해야 함

**단점:**

### 옵션 3: 햄버거 메뉴 + 서랍형 사이드바

```

┌───┬────────────────────────────────────┐
│ ≡ │ Header                             │
├───┴────────────────────────────────────┤
│                                        │
│          Main Content                  │
│                                        │
└────────────────────────────────────────┘

클릭 시:
┌─────────────┬──────────────────────────┐
│ 🏠 홈       │ Header               [X] │
│             ├──────────────────────────┤

│  · 룰렛     │   Main Content           │
│  · 사다리   │                          │
│             │                          │
│ 💻 개발자   │                          │
└─────────────┴──────────────────────────┘
```

**장점:**
- 필요할 때만 표시하여 공간 절약
- 모바일과 데스크톱 동일한 UX
- 깔끔한 인터페이스

**단점:**
- 항상 한 번 더 클릭해야 함
- 현재 위치 파악이 어려울 수 있음

## 🎯 최종 추천: 하이브리드 방식

### 데스크톱 (1024px 이상)
- **축소형 사이드바** (항상 표시)
  - 너비: 60-80px
  - 카테고리 아이콘만 표시
  - 호버 시 도구 목록 팝업 표시

### 태블릿/모바일 (1024px 미만)
- **햄버거 메뉴** (헤더에 버튼)
  - 클릭 시 전체화면 또는 오버레이 메뉴

## 📐 상세 디자인

### 1. 축소형 사이드바 (데스크톱)

```
┌───┬─────────────────────┐
│🏠 │ Header              │
├───┼─────────────────────┤
│🎮 │                     │
│   │                     │
│💻─┼→ [팝업]            │
│   │  💻 개발자 도구     │
│🔄 │  · JSON Parser      │
│   │  · UUID Generator   │
│📝 │  · Timestamp        │
│   │  · Password         │
│🧮 │                     │
│   │  Main Content       │
│   │                     │
│   │                     │
└───┴─────────────────────┘
```

**특징:**
- 너비: 64px
- 배경: `bg-white dark:bg-gray-900`
- 보더: `border-r border-gray-200 dark:border-gray-700`
- 아이콘 크기: 24px
- 호버 시 팝업 표시 (우측으로 확장)

**아이콘 목록:**
- 🏠 홈
- 🎮 게임
- 💻 개발자
- 🔄 변환
- 📝 텍스트
- 🧮 계산기

### 2. 호버 팝업 메뉴

```
┌──────────────────────┐
│ 💻 개발자 도구       │
├──────────────────────┤
│ { } JSON Parser      │
│ 🔑 UUID Generator    │
│ ⏱️ Timestamp         │
│ 🔒 Password          │

```

**스타일:**

- 배경: `bg-white dark:bg-gray-800`
- 그림자: `shadow-xl`
- 둥근 모서리: `rounded-lg`
- 각 항목 호버: `hover:bg-gray-100 dark:hover:bg-gray-700`

**애니메이션:**
- 페이드 인/아웃
- 약간의 슬라이드 효과

## 🎯 기능 요구사항

### 1. 현재 위치 표시
- 현재 페이지에 해당하는 카테고리 및 도구를 하이라이트
- 예: JSON Parser 페이지라면 "개발자" 카테고리와 "JSON Parser" 항목 강조

### 2. 반응형
- 1024px 이상: 축소형 사이드바
- 1024px 미만: 햄버거 메뉴

### 3. 접근성
- 키보드 네비게이션 지원
- ARIA 레이블
- 포커스 표시

### 4. 다크모드 지원
- 라이트/다크 테마에 맞는 색상

## 🎨 색상 및 스타일

### 사이드바
```css
/* 라이트 모드 */
background: white
border: gray-200
text: gray-700
hover: gray-100
active: primary-50 (blue-50)

/* 다크 모드 */
background: gray-900
border: gray-700
text: gray-300
hover: gray-800
active: primary-900 (blue-900)
```

### 현재 항목 하이라이트
```css
background: primary-500 (blue-500)
text: white
icon: white
```

### 호버 팝업
```css
shadow: shadow-xl
border: border-gray-200 dark:border-gray-700
background: white dark:bg-gray-800
```

## 📱 반응형 브레이크포인트

```javascript
// Tailwind breakpoints
sm: 640px   // 모바일 가로
md: 768px   // 태블릿 세로
lg: 1024px  // 태블릿 가로 / 작은 노트북 ← 사이드바 표시 기준
xl: 1280px  // 데스크톱
2xl: 1536px // 큰 화면
```

**네비게이션 전환:**
- `< 1024px`: 햄버거 메뉴
- `≥ 1024px`: 축소형 사이드바

## 🔄 사용자 플로우

### 시나리오 1: 데스크톱에서 도구 변경
1. 사용자가 "JSON Parser" 페이지에 있음
2. 왼쪽 사이드바에서 🎮 (게임) 아이콘에 마우스 호버
3. 팝업에서 게임 도구 목록 표시
4. "룰렛" 클릭
5. 룰렛 페이지로 즉시 이동

### 시나리오 2: 모바일에서 도구 변경
1. 사용자가 "UUID Generator" 페이지에 있음
2. 헤더의 ≡ (햄버거) 버튼 클릭
3. 전체화면 메뉴 열림
4. "게임 / 랜덤" 카테고리 클릭 (펼침)
5. "사다리타기" 클릭
6. 사다리타기 페이지로 이동, 메뉴 자동 닫힘

### 시나리오 3: 홈으로 돌아가기
1. 어느 페이지에서든
2. 사이드바 맨 위 🏠 아이콘 클릭 또는
3. 모바일 메뉴에서 "홈" 클릭
4. 홈 페이지로 이동

## 🎯 기대 효과

1. **사용성 향상**: 어느 페이지에서든 다른 도구로 빠르게 이동
2. **탐색 증가**: 사용자가 더 많은 도구를 발견하고 사용
3. **이탈률 감소**: 도구 간 전환이 쉬워져 사이트 체류 시간 증가
4. **전문성**: 완성도 높은 네비게이션으로 전문적인 인상
